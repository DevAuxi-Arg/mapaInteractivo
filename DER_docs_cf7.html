Arquitectura del
Mapa Interactivo Accesible

Documentación técnica del modelo de datos para una plataforma de navegación educativa interactiva. El sistema permite explorar edificios, pisos y espacios mediante un mapa SVG dinámico, calcular rutas optimizadas con soporte para accesibilidad, y gestionar reportes ciudadanos de barreras arquitectónicas. La autenticación se realiza mediante Firebase Authentication.

profiles
Usuarios y Accesibilidad
📋Propósito

Almacena el perfil extendido de cada usuario registrado. El campo id corresponde directamente al UID generado por Firebase Authentication, vinculando ambos sistemas sin duplicar lógica de autenticación.

🗃️Atributos
PKiduuid
·full_nametext
·avatar_urltext
·roletext · enum
·created_attimestamp
🔗Relaciones
  • →Tiene una accessibility_preferences (1:1, configura)
  • →Puede enviar muchos reports (1:N)
  • →Puede crear routes guardadas (1:N opcional)
💡Uso en la app
  • Autenticación y sesión via Firebase; el UID sincroniza ambas tablas
  • Control de roles: visitor, staff, admin
  • Avatar e identidad del usuario en reportes y preferencias

accessibility_preferences
Usuarios y Accesibilidad
📋Propósito

Configura las necesidades de movilidad y accesibilidad visual de cada usuario. Estos datos guían el algoritmo de ruteo para excluir caminos no aptos y adaptan la interfaz gráfica según las capacidades del usuario.

🗃️Atributos

FKprofile_iduuid
·requires_elevatorboolean
·avoids_stairsboolean
·requires_rampboolean
·high_contrastboolean
·font_size_scaleint (50–200)
·reduced_motionboolean
·updated_attimestamp
🔗Relaciones
  • →Pertenece a un único profiles (1:1)
💡Uso en la app
  • Filtra aristas (edges) no accesibles al calcular rutas

  • Desactiva animaciones SVG cuando reduced_motion = true

Representa cada punto del grafo de navegación posicionado en el SVG. Un nodo puede ser una intersección de pasillos, una entrada, un ascensor, una escalera, una rampa o cualquier punto de interés navegable. Opcionalmente puede corresponder a un espacio físico.

buildings
Mapa y Espacios Físicos
📋Propósito

Representa cada edificio físico de la institución educativa. Es la entidad de más alto nivel en la jerarquía del mapa. Múltiples edificios pueden existir en el mismo campus y cada uno agrupa sus propios pisos y mapas SVG.

🗃️Atributos
PKiduuid
·nametext
·descriptiontext
·short_codetext · ej: "A", "B"
·addresstext
·created_attimestamp
🔗Relaciones
  • →Tiene muchos floors (1:N)
💡Uso en la app
  • Menú de selección inicial: el usuario elige un edificio para explorar
  • El short_code se usa para etiquetar edificios en el mapa general del campus
  • Base para escalar a múltiples sedes o campus en el futuro

FKbuilding_iduuid
·nametext

  • →Pertenece a un buildings (N:1)

💡Uso en la app

  • El svg_viewbox garantiza una escala correcta para los nodos superpuestos
  • Permite navegación entre pisos con transiciones animadas en la UI

Rutas y Recorridos

Rutas calculadas y su secuencia de nodos y aristas para renderizado SVG

2 tablas
🗺️
routes
Rutas y Recorridos
📋Propósito

Representa una ruta calculada y opcionalmente guardada entre dos nodos del grafo. Almacena el resultado del algoritmo de ruteo: peso total, tiempo estimado y si la ruta es apta para usuarios con necesidades de accesibilidad.

🗃️Atributos
PKiduuid
·nametext
FKorigin_node_iduuid
FKdestination_node_iduuid
·is_accessible_routeboolean
·total_weightfloat
·estimated_secondsint
FKcreated_byuuid · nullable
·created_attimestamp
🔗Relaciones
  • →Referencia dos nodes: origen y destino
  • →Tiene muchos route_steps (1:N)
  • →Creada opcionalmente por un profiles
💡Uso en la app
  • Persiste rutas calculadas para compartir o consultar historial
  • El total_weight y estimated_seconds se muestran al usuario antes de iniciar
  • Distingue rutas accesibles de rutas estándar para el modo accesibilidad
  • Base para pre-calcular y cachear rutas frecuentes
👣
route_steps
Rutas y Recorridos
📋Propósito

Secuencia ordenada de nodos y aristas que componen una ruta. Cada paso registra qué nodo se visita y por qué arista se llegó. Esto permite reconstruir el recorrido completo y animarlo en el SVG paso a paso.

🗃️Atributos
PKiduuid
FKroute_iduuid
FKnode_iduuid
FKedge_iduuid · nullable
·step_orderint
🔗Relaciones
  • →Pertenece a una routes (N:1)
  • →Referencia un nodes y opcionalmente una edges
💡Uso en la app
  • Dibuja el recorrido en el SVG: cada edge.svg_path se encadena en orden
  • Genera instrucciones de navegación: "Gira a la derecha en nodo X"
  • Permite avanzar o retroceder pasos en la guía paso a paso

reports
Reportes y Notificaciones
📋Propósito

Centraliza los reportes de problemas enviados por usuarios registrados. Un reporte puede asociarse a un espacio, a una arista del grafo o directamente a una barrera conocida. Los administradores gestionan su estado y prioridad.

🗃️Atributos
PKiduuid
FKreporter_iduuid
FKspace_iduuid · nullable
FKedge_iduuid · nullable
FKbarrier_iduuid · nullable
·report_typetext · enum
·descriptiontext
·statustext · enum
·priorityint (1–5)
·created_at / resolved_attimestamp
🔗Relaciones
  • →Lo envía un profiles (N:1)
  • →Puede referenciar spaces, edges o barriers
  • →Tiene muchas report_photos (1:N)
💡Uso en la app
  • Usuarios reportan "rampa bloqueada", "ascensor fuera de servicio", etc.
  • Los administradores priorizan y resuelven reportes desde el panel
  • Al resolverse un reporte, puede actualizar automáticamente la barrera asociada
  • El mapa muestra alertas visuales en zonas con reportes activos
📸
